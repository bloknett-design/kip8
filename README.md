# КИП ИОС — Техническая документация

## Общие сведения

**Название:** КИП ИОС (Контроль · Измерения · Автоматизация)  
**Тип:** Progressive Web Application (PWA)  
**Назначение:** Инженерные расчёты в области контрольно-измерительных приборов и автоматизации  
**Целевая аудитория:** Инженеры КИП и А, наладчики АСУ ТП, метрологи  
**Язык интерфейса:** Русский  
**Поддержка тем:** Тёмная (по умолчанию) / Светлая  

---

### sw.js (5 KB, 154 строки)
**Роль:** Service Worker для офлайн-работы и кэширования.

**Стратегии кэширования:**
- `Cache First` — Статические ресурсы (CSS, JS, HTML, иконки)
- `Network First` — Внешние ресурсы (OneDrive-ссылки, CDN)
- `offline.html` — Fallback при отсутствии сети

**Жизненный цикл:**
1. `install` → Кэширование static assets
2. `activate` → Очистка старых кэшей, `clients.claim()`
3. `fetch` → Выбор стратегии по типу запроса
4. `message` → Обработка команд (SKIP_WAITING)

## Функциональные модули

### 1. Конвертер единиц

**Файл:** `app.js`, секция `unitData`

**Категории:**
- Давление: Па, кПа, МПа, бар, мбар, атм, кгс/см², мм рт.ст., PSI, мм вод.ст.
- Расход: м³/ч, м³/с, л/мин, л/с, т/ч, кг/с, кг/ч
- Масса: кг, г, мг, т, фунт, унция
- Температура: °C, °F, K, °R, °Ra (с отдельной логикой)
- Длина: м, км, см, мм, мкм, миля, фут, дюйм, ярд
- Плотность: кг/м³, г/см³, г/л, кг/л
- Время: сек, мс, мин, час, сутки, неделя, месяц, год
- Объём: м³, литр, мл, см³, галлон, баррель, куб. фут, дм³

**Формула конвертации:**
```
значение_в_целевой_единице = (входное_значение * factor_входной) / factor_целевой
```

**Особенность температуры:** Использует промежуточное значение в °C, т.к. шкалы имеют разные нули.

---

### 2. Шкала-сигнал

**Файл:** `app.js`, функции: `calcScaleSignal`, `queryFromScale`, `queryFromSignal`

**Типы шкал:**
- `linear` — Линейная: `signal = min + (max-min) * (scale% / 100)`
- `quadratic` — Квадратичная: `signal = min + (max-min) * (scale% / 100)²`
- `sqrt` — Корнеизвлекающая: `signal = min + (max-min) * √(scale% / 100)`

**Предустановленные диапазоны:**
- Шкала: 0-100%, 0-20 мА, 0-10 В, 1-5 В, -10...+10 В, 20-100 кПа и др.
- Сигнал: 4-20 мА, 0-20 мА, 0-10 В, 20-100 кПа и др.

**Обратный расчёт:** Возможен поиск сигнала по значению шкалы и наоборот.

---

### 3. Расчёт погрешностей

**Файл:** `app.js`, функция `calcErrorUniversal(config)`

**Формула (ГОСТ 8.401-80):**
```
Δ = (γ / 100) * Xₙ
где:
  γ — класс точности (%)
  Xₙ — нормирующее значение

Xₙ = |max - min|  (если min = 0 или min*max < 0)
Xₙ = |max|         (если min ≠ 0 и знаки одинаковые)
```

**Типы приборов:**
- Датчик давления — МПа, бар, кПа
- Термопреобразователи сопротивления (ТС) — Pt, Cu, Ni
- Термопары (ТП) — K, J, T, N, E, R, S, B
- Расходомер — м³/ч, л/мин, т/ч
- Уровнемер — мм, %
- Аналоговый преобразователь — 4-20 мА
- Комплект приборов — суммарная погрешность √(Δ₁² + Δ₂² + ...)

**ТС — ГОСТ 6651-2009:**
- Группы: Pt (платиновые), Cu (медные), Ni (никелевые)
- Классы допуска: AA, A, B, C
- Формула: `Δ = ±(const + coeff * |t|)` °C

**ТП — ГОСТ Р 8.585-2001:**
- Классы точности: 1, 2, 3
- Формулы по диапазонам: константа, линейная, линейная²

### 4. Буйковый уровнемер

**Файл:** `app.js`, функция `calculateBuoyCalibration()`

**Методы калибровки:**
- **С буйком:** `m_гири = m_выт + m_подвеса`
- **Без буйка:** `m_гири = m_буйка - m_выт + m_подвеса`

**Формулы:**
```
V_буйка = π * (D/2)² * L          # Объём буйка
m_выт = V_погруж * ρ_жидкости      # Выталкивающая сила (масса)
```

**Предустановленные жидкости:**
- Вода (998 кг/м³)
- Дивинилбензол (926)
- Диэтилбензол (860)
- Стирол (906)
- Дихлорэтан (1253)
- Этиловый спирт (789)
- Этиленгликоль (1111.5)
- Едкий натр (2130)
- Кальцинированная сода (2530)
- Олеум (1890)
- Серная кислота 96% (1834)
- Динитрофенол (1683)
- Перекись бензоила (1334)
- Перекись водорода 100% (1450)

**Выходной сигнал:** 4-20 мА, 0-10 В, 20-100 кПа и др. (с кастомным диапазоном)

### 5. Датчики температуры

**Файл:** `app.js`, функции: `calcTempSensor`, `calcRtdResistance`, `calcCuResistance`, `calcTcVoltage`

**Термометры сопротивления (ТС):**

| Тип | R₀ | α | НСХ |
|-----|-----|-----|-----|
| Cu50 (50М) | 50 Ом | 0.00428 | W₁₀₀ = 1.428 |
| Cu100 (100М) | 100 Ом | 0.00428 | W₁₀₀ = 1.428 |
| Pt50 (50П) | 50 Ом | 0.00391 | W₁₀₀ = 1.391 |
| Pt100 (100П) | 100 Ом | 0.00391 | W₁₀₀ = 1.391 |
| Pt100 (IEC) | 100 Ом | 0.00385 | W₁₀₀ = 1.385 |
| Pt1000 | 1000 Ом | 0.00385 | W₁₀₀ = 1.385 |

**Формулы ТС:**
```
Cu:  R(t) = R₀(1 + α·t)
Pt:  R(t) = R₀(1 + A·t + B·t²)                    # t ≥ 0
     R(t) = R₀(1 + A·t + B·t² + C(t-100)t³)       # t < 0 (IEC)
```

**Термопары (ТП):**

| Тип | Диапазон (кл.2) | Название |
|-----|-----------------|----------|
| K | -40...1300°C | ТХА (хромель-алюмель) |
| J | 0...900°C | ТЖК (железо-константан) |
| T | -40...350°C | ТМК (медь-константан) |
| N | -40...1300°C | ТНН (нисил-нисил) |
| E | -40...900°C | ТХКн (хромель-константан) |
| R | 0...1600°C | ТПП (платина-родий) |
| S | 0...1600°C | ТПП (платина-родий) |
| B | 600...1700°C | ТПР (платина-родий) |

**Расчёт ЭДС:** Полиномиальная аппроксимация по НИСТ:
```
E(t) = Σ cᵢ · tⁱ   (i = 0...n)
```

Коэффициенты загружаются асинхронно из `nist_coefficients.json`.

### 6. Автоматический выключатель

**Файл:** `app.js`, функция `calcCircuitBreaker()`

**Нормативы:** ГОСТ Р 50345-2010, ПУЭ

**Типы питания:**
- Переменный ток: ~220 В, ~24 В, ~36 В
- Постоянный ток: =24 В, =12 В, =36 В
- Кастомное напряжение

**Типы нагрузки:**
- По мощности (Вт)
- По току (А)

**Профили потребителей:**

| Профиль | cos φ | Кпуск | Описание |
|---------|-------|-------|----------|
| КИП активная | 1.0 | 1.0 | Датчики, индикация |
| КИП индуктивная | 0.7 | 3.0 | Клапана, реле, соленоиды |
| АСУ ТП контроллер | 0.95 | 1.2 | Контроллер + модули |
| АСУ ТП ИМ | 0.75 | 5.0 | Исполнительные механизмы |
| Смешанная | 0.85 | 2.0 | Комбинация |

**Характеристики срабатывания:**
- B (3-5 In) — электроника, датчики
- C (5-10 In) — клапана, реле (по умолчанию)
- D (10-20 In) — соленоиды, пускатели
- Z (2-3 In) — сверхчувствительная защита

**Расчёт:**
```
Iр = P / (U · cos φ) · 1.25    # с запасом 25%
Iпуск = Iр · Кпуск
Iном_автомата = nearestStandard(Iр · 1.25)
```

**Проверка сечения кабеля:**

| Сечение | Материал | Imax | Применение |
|---------|----------|------|------------|
| 0.75 мм² | Cu | 10 А | Сигнальные цепи, 4-20 мА |
| 1.0 мм² | Cu | 10 А | Питание 24 В DC |
| 1.5 мм² | Cu | 16 А | Питание 220 В |
| 2.5 мм² | Cu | 25 А | Группа приборов |

### 7. Сужающее устройство (диафрагма)

**Файл:** `app.js`, функции: `calcOrificeDp`, `calcOrificeFlow`, `calcOrificeDiameter`

**Нормативы:** РД 50-411-83, ГОСТ 26969-86

**Типы СУ:**
- Диафрагма стандартная
- Сопло стандартное
- Труба Вентури

**Типы сред:**
- Жидкость (несжимаемая, ε = 1)
- Газ/пар (сжимаемая, ε < 1)

**Формулы:**

**Расход через СУ:**
```
Q = α · ε · (πD²/4) · √(2Δp/ρ) · m

где:
  α — коэффициент расхода (зависит от m и типа СУ)
  ε — коэффициент расширения (для газов)
  m = (d/D)² — относительное отверстие
  D — диаметр трубы
  d — диаметр отверстия
```

**Коэффициент расхода α:**
```
α = A + B·m + C·m² + D·m³
```

Коэффициенты по типам СУ:

| Тип | A | B | C | D |
|-----|------|--------|--------|-------|
| Диафрагма | 0.5959 | 0.0312 | -0.1840 | 0.0390 |
| Сопло | 0.9965 | 0.0105 | -0.0420 | 0.0085 |
| Вентури | 0.9840 | 0.0050 | -0.0150 | 0.0030 |

**Коэффициент расширения ε (газы):**
```
ε = 1 - (0.41 + 0.35·m^1.5) · (Δp/P) / κ

где:
  κ — показатель адиабаты
  P — абсолютное давление
```

**Тепловое расширение:**
```
D(t) = D₂₀ · (1 + α_материала · (t - 20))
```

Коэффициенты линейного расширения:

| Материал | α, 1/°C |
|----------|---------|
| Сталь 20 | 11.9·10⁻⁶ |
| 12Х18Н10Т | 16.6·10⁻⁶ |
| 15Х5М | 11.5·10⁻⁶ |
| Латунь | 18.9·10⁻⁶ |
| Бронза | 17.6·10⁻⁶ |

**Расчётные режимы:**
1. **Перепад Δp** — по Q и d₂₀
2. **Расход Q** — по Δp и d₂₀
3. **Диаметр d₂₀** — по Q и Δp (итерационный метод, до 50 итераций)

**Проверки:**
- m ∈ [0.05, 0.64]
- ReD ≥ 10⁵
- Δp ≤ 0.25·P (для газов)
- d₂₀/D₂₀ ≤ 0.8

## API и утилиты

### SafeStorage

Безопасная обёртка над `localStorage` с fallback в память.

```javascript
SafeStorage.setItem(key, value)    // Сохранить
SafeStorage.getItem(key)           // Получить (null если нет)
SafeStorage.removeItem(key)        // Удалить
SafeStorage.clear()                // Очистить всё
SafeStorage.isAvailable()          // Проверить доступность
```

**Сценарии защиты:**
- Приватный режим Safari (QuotaExceededError)
- Переполнение хранилища
- Отключённый localStorage

### Validate

Валидация входных данных для расчётов.

```javascript
Validate.isPositiveNumber(value, name)     // > 0
Validate.isNumber(value, name)              // любое число
Validate.isValidRange(min, max, name)       // min < max
Validate.isPercentage(value, name)          // 0-100%
Validate.validateFields({key: check, ...})  // массовая проверка
```

**Возвращает:** `{valid: true, value: number}` или `{valid: false, error: string}`

### Распространённые проблемы

| Проблема | Причина | Решение |
|----------|---------|---------|
| SW не регистрируется | Не HTTPS или localhost | Запускать через HTTPS |
| Кэш не обновляется | Старый SW активен | Закрыть все вкладки и открыть заново |
| localStorage падает | Приватный режим | SafeStorage автоматически переключается на память |
| NaN в расчётах | Невалидный ввод | Validate.validateFields показывает toast |
| Тема не сохраняется | localStorage недоступен | Проверить SafeStorage.isAvailable() |
